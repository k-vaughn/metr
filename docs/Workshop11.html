METR Deployment Part I

Questions for METR Workshop 11: METR Deployment Part I

The eleventh METR workshop will include focus on the following questions:

  • What prerequisites should be met for METR participation?
  • What events might trigger the revocation of authorization to serve as a METR-affiliated regulator, collector, disseminator, or user (e.g., field reporter)?
  • Should we define standardized conformance or maturity capabilities for METR-systems?
  • What features of METR need to be implemented by a system to claim basic METR conformance?
  • Is there a need to indicate the multiple regulators that might exist within a single jurisdictional entity?
  • How should user systems (e.g., ADS-equipped vehicles, driver support systems) respond in the absence of real-time METR data for the user's current location?
  • What complications arise if only a subset of vehicles have real-time data?
  • Is there a greater need for METR availability in areas that support dynamic rules?
  • To what extent and for what purpose(s) is there a need to track the performance of each METR component?
  • To what extent and for what purpose(s) is there a need to monitor the accuracy of the electronic rules being distributed?
  • To what extent is there a need to support non-repudiation (e.g., to support auditing, insurance claims)?
  • What management and control features does METR need to support?
  • Who might be responsible for maintaining the operation of the METR subsystems?
  • How is each entity funded?
